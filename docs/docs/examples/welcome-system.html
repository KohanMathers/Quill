Example — Welcome System

Example — Welcome System

This example shows a small, practical welcome system written in Quill. It greets players when they join, gives a small starter kit, and exposes a toggle command for server operators. The script demonstrates event handlers, built-in functions, and a persistent flag.

The script

Save this as welcome.ql (or any name) and load it with the Quill plugin.

let welcome_enabled = true;   // persistent flag

// Give a short welcome and starter kit on join
OnEvent(PlayerJoin) {
    if !welcome_enabled {
        return;
    }

    sendmessage(player, "Welcome to the server, {player.name}!");
    give(player, "minecraft:bread", 5);
    sendtitle(player, "Welcome!", "Enjoy your stay, {player.name}", 10, 70, 20);
}

// OPs can toggle the welcome system
OnEvent(PlayerChat) {
    if chat.message == "!toggle_welcome" && is_op(player) {
        welcome_enabled = !welcome_enabled;
        sendmessage(player, "Welcome system: {welcome_enabled}");
    }
}

Notes
- welcome_enabled is a root-level variable and accessible in all subscopes.
- OnEvent(PlayerJoin) runs with a player context. Built-in functions used: sendmessage, give, sendtitle.
- is_op(player) checks operator status before allowing the toggle.

Load & run

Place welcome.ql in the location used by your Quill plugin (your server's scripts directory) and run:

/quill load welcome.q

Watch the server console for parser errors.

Test steps

  1. Join the server with a non-OP account — you should receive the welcome message, 5 bread, and the title.
  2. With an OP account, enter !toggle_welcome and confirm the plugin replies Welcome system: false (or true).
  3. Toggle back and re-test with another non-OP account to confirm behavior.

Extensions & improvements

  • Scope-limited welcome: only welcome players who join inside a Scope region using in_region(player, my_scope).
  • Richer starter kit: give armor, tools, or items using give(player, "minecraft:iron_sword") and give(player, item("minecraft:shield", 1)).

Troubleshooting

  • Script fails to load: ensure the file is placed where Quill expects scripts and check server logs for parser errors.
  • Built-in functions not working: verify the plugin version and server compatibility (Minecraft server type and Java version).
