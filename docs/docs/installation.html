Installation

Installation

This guide shows you how to install and configure Quill on your Minecraft server.

Requirements

  • Minecraft Server: Bukkit, Spigot, or Paper (1.20+)
  • Java: 21 or higher
  • Dependencies: None required

Installation Steps

1. Download Quill

Download the latest Quill-1.0.0.jar from one of the following providers.
- Modrinth
- Hangar
- SpigotMC
- Bukkit/Curseforge
- PolyMart

2. Install the Plugin

  1. Place Quill-1.0.0.jar in your server's plugins/ folder
  2. Restart your server (or use /reload confirm if your server supports it)
  3. Quill will generate its configuration files automatically

3. Verify Installation

Run the following command in-game or console:

/quill info

You should see output showing:
- Quill version
- Number of loaded scripts
- Scripts directory path

Configuration

After first startup, Quill creates a config.yml file in plugins/Quill/:

###############################################
#      Quill – A sandboxed scripting language #
#          for Minecraft event automation     #
#               By Kohan Mathers              #
###############################################

##############
# Web Editor #
##############

editor:
  url: https://quill.kmathers.co.uk/editor/#

####################
# DO NOT CHANGE THESE
config-version: 1
translation-version: 1

Editor URL

The editor.url setting points to the web-based script editor. When you run /quill edit <filename>, Quill creates a session and provides you with a clickable link to this editor.

Default: https://quill.kmathers.co.uk/editor/#

You can host your own editor instance if desired, but the default should work for most users.

Directory Structure

After installation, your plugin folder will look like this:

plugins/Quill/
├── config.yml              # Plugin configuration
├── translations.yml        # Localization strings
├── scripts/                # Script storage
│   ├── example.ql         # Global scripts
│   └── my-scope/          # Scope-specific scripts
│       └── game.ql
└── scopes/                # Scope configurations
    └── my-scope.yml

Permissions

Quill uses Bukkit's permission system. Here are the key permissions:

Admin Permissions

quill.admin              # Full access (all permissions below)
quill.scope.create       # Create new scopes
quill.scope.delete.any   # Delete any scope
quill.script.load.global # Load global scripts
quill.script.edit.global # Edit global scripts

Player Permissions

quill.command            # Use /quill command
quill.script.load.scope  # Load scripts in owned scopes
quill.script.edit.scope  # Edit scripts in owned scopes
quill.scope.list         # List your own scopes
quill.scope.info         # View your scope info

See permissions.html for the complete list.

First Steps

For Server Admins

  1. Create a test scope:
    /quill scope create test-zone YourUsername 0 60 0 50 100 50 whitelist

  2. Grant basic permissions to the scope:
    /quill scope permission grant test-zone sendmessage /quill scope permission grant test-zone log

  3. Create a test script:
    /quill edit test-zone/welcome.ql

  4. Write a simple script in the editor:
    quill onEvent(PlayerJoin) { log("Player joined: {player.name}") }

  5. Load the script:
    /quill load test-zone/welcome.ql

For Players

If your server admin has given you a scope:

  1. Check your scopes:
    /quill scope list

  2. View scope details:
    /quill scope info your-scope-name

  3. Create a script:
    /quill edit your-scope-name/my-script.ql

  4. Load your script:
    /quill load your-scope-name/my-script.ql

Troubleshooting

Plugin won't load

Check your Java version:

java -version

Quill requires Java 21 or higher.

Check server logs:
Look for errors in logs/latest.log. Common issues:
- Incompatible Minecraft version
- Missing dependencies (shouldn't happen with Quill)
- File permission errors

Scripts won't load

Verify file location:
Scripts must be in plugins/Quill/scripts/ (for global) or plugins/Quill/scripts/scope-name/ (for scoped scripts).

Check file extension:
Scripts must end in .ql or .quill.

Review permissions:
Make sure your scope has permission for the functions you're using:

/quill scope permission list your-scope-name

Editor link doesn't work

Check the editor URL:
Verify editor.url in config.yml starts with http:// or https://.

Firewall issues:
Some server hosts block outgoing connections. Try accessing the editor manually in your browser.

Permission errors

Global scripts:
You need quill.script.load.global permission to load scripts in the root scripts/ folder.

Scoped scripts:
You need either:
- quill.script.load.scope permission AND ownership of the scope, OR
- quill.script.load.global permission

Updating Quill

  1. Backup your data:
    plugins/Quill/scripts/ plugins/Quill/scopes/

  2. Replace the JAR:
    - Stop your server
    - Replace Quill-X.X.X.jar with the new version
    - Start your server

  3. Check for breaking changes:
    Review the changelog for any breaking changes.

Quill will automatically migrate your configuration if needed.
