Quill/Language/Data Types

Data Types

Data Types

Overview

Quill supports a variety of data types, from simple numbers and strings to complex Minecraft objects like players and locations. Understanding these types is essential for writing effective scripts.

Type Categories:

  • Primitive Types - Basic values: Number, String, Boolean, Null
  • Collection Types - Containers for multiple values: List, Map
  • Minecraft Types - Game objects: Player, Location, Item, Entity, World, Scope, Region, Event

Type System:

Quill is dynamically typed - variables don't have fixed types. The same variable can hold different types of values:

let x = 10        // x is a Number
x = "hello"       // Now x is a String
x = true          // Now x is a Boolean

Type Safety:

While types are dynamic, functions expect specific types. Passing the wrong type causes runtime errors:

let name = "Steve"
teleport(name, location(0, 64, 0))  // Error: expected Player, got String

Primitive Types

Number

Numbers represent numeric values - both integers and decimals.

Integer Numbers

Whole numbers without decimal points:

let count = 0
let score = 100
let negative = -50
let large = 1000000

Integer range:
- Internally stored as doubles (JavaScript-style)
- Safe integer range: -2^53 to 2^53 (approximately ±9 quadrillion)
- Beyond this range, precision may be lost

Common uses:

let player_count = 10
let max_health = 20
let x_coordinate = 128

Floating-Point Numbers

Numbers with decimal points:

let pi = 3.14159
let half = 0.5
let precise = 1.23456789
let scientific = 1.5e10  // Not supported - use: 15000000000

Precision:
- Stored as 64-bit floating point (double precision)
- ~15-17 significant decimal digits
- Floating point arithmetic may have rounding errors

let result = 0.1 + 0.2  // 0.30000000000000004 (floating point quirk)

Common uses:

let health = 15.5
let distance = 10.75
let multiplier = 1.5

Number Operations

All standard arithmetic operations:

let a = 10
let b = 3

// Basic arithmetic
let sum = a + b           // 13
let difference = a - b    // 7
let product = a * b       // 30
let quotient = a / b      // 3.333...
let remainder = a % b     // 1 (modulo)

// Negation
let negative = -a         // -10

// Complex expressions
let result = (a + b) * 2 - b / 2  // 24.5

Special number values:

let result = 10 / 0  // Infinity (not recommended - use conditionals)

Math functions available:

abs(-10)        // 10 (absolute value)
sqrt(16)        // 4 (square root)
pow(2, 3)       // 8 (2^3)
round(3.7)      // 4 (round to nearest)
floor(3.7)      // 3 (round down)
ceil(3.2)       // 4 (round up)

String

Strings represent text - sequences of characters.

String Literals

Create strings with double or single quotes:

let name = "Steve"
let message = 'Hello, World!'
let empty = ""

Multi-line strings are NOT directly supported:

// This does NOT work:
let poem = "Roses are red
Violets are blue"

// Instead, use string concatenation:
let poem = "Roses are red\n" +
           "Violets are blue"

Escape Sequences

Special character sequences inside strings:

let newline = "Line 1\nLine 2"      // Line break
let tab = "Name:\tSteve"             // Tab character
let quote = "He said \"Hello\""      // Escaped quote
let backslash = "Path: C:\\folder"   // Escaped backslash

// Common escape sequences:
// \n - Newline
// \t - Tab
// \r - Carriage return
// \\ - Backslash
// \" - Double quote

Example output:

log("Line 1\nLine 2")
// Console shows:
// Line 1
// Line 2

log("Name:\tSteve")
// Console shows:
// Name:    Steve

String Operations

Concatenation - Join strings with +:

let first = "Hello"
let last = "World"
let greeting = first + " " + last  // "Hello World"

String interpolation - Embed variables (preferred):

let name = "Steve"
let score = 100
let message = "Player {name} scored {score} points!"
// Result: "Player Steve scored 100 points!"

See String Interpolation for full details.

String functions:

let text = "Hello World"

len(text)              // 11 (length)
contains(text, "World") // true (substring check)
split(text, " ")       // ["Hello", "World"] (split into list)

let words = ["Hello", "World"]
join(words, " ")       // "Hello World" (join list into string)

Comparison:

"apple" == "apple"     // true
"apple" == "Apple"     // false (case-sensitive)
"apple" != "orange"    // true

Boolean

Booleans represent truth values - either true or false.

Boolean Values

Only two possible values:

let is_active = true
let is_paused = false

let is_winner = score > 100
let has_permission = is_online(player) && is_op(player)

Logical operations:

let a = true
let b = false

a && b    // false (AND - both must be true)
a || b    // true  (OR - at least one must be true)
!a        // false (NOT - inverts the value)

From comparisons:

let is_high_score = score >= 100
let is_low_health = health < 5
let is_equal = x == y

Truthy and Falsy Values

Non-boolean values can be used in boolean contexts. They're automatically converted:

Falsy values (convert to false):

false          // Obviously false
null           // Null is falsy
0              // Zero is falsy
""             // Empty string is falsy
[]             // Empty list is falsy

Truthy values (convert to true):

true           // Obviously true
1              // Any non-zero number
-1             // Including negative
"text"         // Any non-empty string
" "            // Even just whitespace
[1]            // Any non-empty list

Examples in conditions:

let text = ""
if text {
    log("Has text")    // Doesn't run - empty string is falsy
}

let count = 0
if count {
    log("Has items")   // Doesn't run - zero is falsy
}

let players = []
if players {
    log("Has players") // Doesn't run - empty list is falsy
}

Explicit conversion recommended:

// Implicit (works but unclear)
if text {
    // ...
}

// Explicit (clearer intent)
if len(text) > 0 {
    // ...
}

Null

Null represents the absence of a value.

Null Values

Use null to indicate "no value":

let player = null           // No player yet
let location = null         // No location set
let cached_value = null     // Cache is empty

Common uses:

// Optional values
let winner = null
if score > 100 {
    winner = player
}

// Uninitialized state
let target_player = null
// ... later ...
target_player = get_player("Steve")

// Reset values
let cached_data = expensive_calculation()
// ... use cached_data ...
cached_data = null  // Clear the cache

Checking for Null

Always check for null before using values:

let player = get_player("Steve")

// Bad - will error if player is null
sendmessage(player, "Hello")

// Good - check first
if player != null {
    sendmessage(player, "Hello")
}

// Also good - null is falsy
if player {
    sendmessage(player, "Hello")
}

Null in comparisons:

null == null       // true
null == false      // false
null == 0          // false
null == ""         // false

let x = null
x == null          // true
x != null          // false

Functions returning null:

let player = get_player("NonexistentPlayer")  // Returns null
let region = get_region(my_scope)             // Returns null if no region

// Always handle null returns
if player == null {
    log("Player not found")
    return
}

Collection Types

List

Lists store ordered sequences of values.

Creating Lists

Use square brackets with comma-separated values:

let empty = []
let numbers = [1, 2, 3, 4, 5]
let names = ["Steve", "Alex", "Herobrine"]
let mixed = [1, "text", true, null]  // Can mix types

// Multi-line for readability
let items = [
    "diamond_sword",
    "diamond_pickaxe",
    "diamond_axe"
]

Accessing Elements

Use square brackets with an index (0-based):

let names = ["Steve", "Alex", "Herobrine"]

let first = names[0]     // "Steve"
let second = names[1]    // "Alex"
let third = names[2]     // "Herobrine"

// Negative indices NOT supported
let last = names[-1]     // Error

Out of bounds:

let names = ["Steve", "Alex"]
let item = names[10]     // Returns null (not an error)

Assignment:

let names = ["Steve", "Alex"]
names[0] = "Notch"       // Modifies the list
// names is now ["Notch", "Alex"]

List Operations

Length:

let items = [1, 2, 3]
let count = len(items)   // 3

Adding elements:

let items = [1, 2, 3]
append(items, 4)         // items is now [1, 2, 3, 4]
append(items, 5)         // items is now [1, 2, 3, 4, 5]

Removing elements:

let items = [1, 2, 3, 4, 5]
remove(items, 2)         // Remove at index 2
// items is now [1, 2, 4, 5]

Checking contents:

let items = ["sword", "bow", "arrow"]
contains(items, "bow")          // true
contains(items, "diamond")      // false

Creating ranges:

range(0, 5)              // [0, 1, 2, 3, 4]
range(5, 10)             // [5, 6, 7, 8, 9]
range(10, 5)             // [10, 9, 8, 7, 6] (descending)

Random selection:

let items = ["sword", "bow", "axe"]
let random_item = random_choice(items)

Concatenation:

let list1 = [1, 2]
let list2 = [3, 4]
let combined = list1 + list2  // [1, 2, 3, 4]

Iterating Over Lists

Use for loops:

let items = ["sword", "bow", "arrow"]

for item in items {
    log("Item: {item}")
}

// With indices (using range)
for i in range(0, len(items)) {
    log("Index {i}: {items[i]}")
}

Map

Maps store key-value pairs (like dictionaries or objects).

Creating Maps

Use curly braces with key-value pairs:

let empty = {}

let player_data = {
    name: "Steve",
    score: 100,
    level: 5
}

let config = {
    max_players: 10,
    pvp_enabled: true,
    spawn_x: 0,
    spawn_y: 64,
    spawn_z: 0
}

Keys must be identifiers or strings:

// With identifier keys (no quotes needed)
let data = {
    name: "Steve",
    age: 25
}

// With string keys (quotes required for special characters)
let data = {
    "player-name": "Steve",
    "high score": 100
}

Values can be any type:

let complex = {
    number: 42,
    text: "Hello",
    flag: true,
    list: [1, 2, 3],
    nested: {
        inner: "value"
    }
}

Accessing Values

Use dot notation or bracket notation:

let player = {
    name: "Steve",
    score: 100
}

// Dot notation (preferred)
let name = player.name       // "Steve"
let score = player.score     // 100

// Bracket notation (for dynamic keys)
let key = "name"
let value = player[key]      // "Steve"

Non-existent keys return null:

let player = {name: "Steve"}
let age = player.age         // null (key doesn't exist)

Map Operations

Setting values:

let player = {name: "Steve"}

// Add new key
player.score = 100
// player is now {name: "Steve", score: 100}

// Update existing key
player.name = "Alex"
// player is now {name: "Alex", score: 100}

Checking for keys:

let player = {name: "Steve", score: 100}

if player.age != null {
    log("Player has age")
}

Nested access:

let data = {
    player: {
        name: "Steve",
        stats: {
            health: 20,
            hunger: 15
        }
    }
}

let health = data.player.stats.health  // 20

Iterating Over Maps

Maps are NOT directly iterable in Quill. Access known keys:

let scores = {
    steve: 100,
    alex: 150,
    herobrine: 200
}

// Access known keys
log("Steve: {scores.steve}")
log("Alex: {scores.alex}")
log("Herobrine: {scores.herobrine}")

// Or use a list of keys
let keys = ["steve", "alex", "herobrine"]
for key in keys {
    log("{key}: {scores[key]}")
}

Minecraft Types

These types represent objects in the Minecraft world.

Player

Represents a player on the server.

Player Properties

Access player information with dot notation:

let player = get_player("Steve")

player.name          // "Steve" (player's username)
player.health        // 20.0 (current health)
player.hunger        // 20 (food level)
player.location      // Location object
player.gamemode      // "survival" (gamemode name)
player.flying        // true/false (is flying)
player.online        // true/false (is online)

Example:

OnEvent(PlayerJoin) {
    log("Player {player.name} joined")
    log("Health: {player.health}")
    log("Location: {player.location}")
}

Player Methods

Player objects work with player functions:

let player = get_player("Steve")

// Movement
teleport(player, location(0, 64, 0))

// Inventory
give(player, "diamond_sword")
remove_item(player, "dirt", 64)
let has_diamonds = has_item(player, "diamond", 5)

// State
set_health(player, 20)
set_hunger(player, 20)
heal(player)
set_gamemode(player, "creative")

// Communication
sendmessage(player, "Hello!")
playsound(player, "entity.player.levelup", 1.0, 1.0)

// Getters
let health = get_health(player)
let location = get_location(player)
let name = get_name(player)

Location

Represents a position in 3D space.

Location Properties

let loc = location(100, 64, 200)

loc.x        // 100.0 (X coordinate)
loc.y        // 64.0 (Y coordinate)
loc.z        // 200.0 (Z coordinate)
loc.world    // World object

Example:

let player_loc = get_location(player)
log("Player at: {player_loc.x}, {player_loc.y}, {player_loc.z}")

Creating Locations

Use the location() constructor:

// With default world
let spawn = location(0, 64, 0)

// With specific world
let nether_spawn = location(0, 64, 0, "world_nether")

// From player
let player_pos = get_location(player)

// From event
OnEvent(BlockBreak) {
    let block_pos = block.location
}

Using locations:

let spawn = location(0, 64, 0)

// Teleport players
teleport(player, spawn)

// Set blocks
set_block(spawn, "diamond_block")

// Get blocks
let block_type = get_block(spawn)

// Calculate distance
let dist = distance(spawn, player.location)

Item

Represents an item stack.

Item Properties

let item = item("diamond_sword", 1)

item.type        // "diamond_sword" (item material)
item.amount      // 1 (stack size)

Example:

OnEvent(PlayerDropItem) {
    log("Dropped: {item.type} x{item.amount}")
}

Creating Items

Use the item() constructor:

// Just type (default amount: 1)
let sword = item("diamond_sword")

// Type and amount
let arrows = item("arrow", 64)

// With namespace
let custom = item("minecraft:golden_apple")

Using items:

let sword = item("diamond_sword")
give(player, sword)

let apple = item("apple", 5)
give(player, apple)

Entity

Represents a non-player entity (mob, animal, etc.).

Entity Properties

OnEvent(EntitySpawn) {
    log("Entity type: {entity.type}")
    log("Location: {entity.location}")
    log("Is alive: {entity.alive}")
}

Properties:

entity.type          // "zombie", "cow", "creeper", etc.
entity.location      // Location object
entity.alive         // true/false (is still alive)

Using entities:

// Spawn entity
let zombie = spawn_entity("zombie", location(0, 64, 0))

// Remove entity
remove_entity(zombie)

// Check type
if zombie.type == "zombie" {
    remove_entity(zombie)
}

World

Represents a Minecraft world dimension.

World Properties

let world = get_world("world")

world.name           // "world" (world name)

// No other properties directly accessible
// Use functions instead

Using worlds:

let world = get_world("world")

// Time control
set_time(world, 0)           // Morning
let current_time = get_time(world)

// Weather control
set_weather(world, "rain", 6000)
let weather = get_weather(world)

Scope

Represents a Quill scope (physical region with scripts).

Scope Properties

let my_scope = new Scope(-50, 60, -50, 50, 120, 50)

my_scope.players     // List of players in scope
my_scope.region      // Region object (boundaries)

// Access variables (if scope is parent)
my_scope.some_variable

Using scopes:

let arena = new Scope(-50, 60, -50, 50, 120, 50)

// Player management
addtoscope(player, arena)
removefromscope(player, arena)
let players = getplayers(arena)

// Region checking
if in_region(player, arena) {
    log("Player in arena")
}

// Iterate players
for p in arena.players {
    heal(p)
}

Region

Represents a 3D box area (scope boundaries).

Region Properties

let region = get_region(my_scope)

region.x1, region.y1, region.z1    // First corner
region.x2, region.y2, region.z2    // Opposite corner

// If region is null, scope has no region
if region == null {
    log("No region defined")
}

Using regions:

let region = get_region(arena)

if region != null {
    log("Boundaries:")
    log("  ({region.x1}, {region.y1}, {region.z1})")
    log("  to")
    log("  ({region.x2}, {region.y2}, {region.z2})")
}

Event

Represents a Minecraft event that occurred.

Event Properties

Event objects are provided in event handlers and contain the original Bukkit event:

OnEvent(PlayerMove) {
    // Cancel the event
    cancel(event)
}

OnEvent(BlockBreak) {
    // Event object available but properties accessed via context
    // (block, player variables are provided separately)
}

Using events:

OnEvent(PlayerInteract) {
    // Cancel certain interactions
    if block.type == "tnt" {
        cancel(event)
        sendmessage(player, "TNT is disabled!")
    }
}

OnEvent(BlockPlace) {
    // Prevent placing in certain areas
    if in_region(block.location, protected_zone) {
        cancel(event)
    }
}

Type Checking

Check what type a value is:

Using type_of():

let x = 10
type_of(x)           // "number"

let text = "hello"
type_of(text)        // "string"

let flag = true
type_of(flag)        // "boolean"

let nothing = null
type_of(nothing)     // "null"

let items = [1, 2, 3]
type_of(items)       // "list"

let player = get_player("Steve")
type_of(player)      // "player"

Available type names:
- "number"
- "string"
- "boolean"
- "null"
- "list"
- "map"
- "player"
- "location"
- "item"
- "entity"
- "world"
- "scope"
- "region"
- "event"
- "function"

Conditional logic based on type:

let value = some_function()

if type_of(value) == "player" {
    heal(value)
} else if type_of(value) == "number" {
    log("Got number: {value}")
} else {
    log("Unknown type: {type_of(value)}")
}

Type Conversion

Convert between types:

To String:

to_string(42)           // "42"
to_string(3.14)         // "3.14"
to_string(true)         // "true"
to_string(null)         // "null"
to_string([1, 2, 3])    // "[1, 2, 3]"

// Alternative: string concatenation
let text = "" + 42      // "42"

To Number:

to_number("42")         // 42.0
to_number("3.14")       // 3.14
to_number(true)         // 1.0
to_number(false)        // 0.0

// Errors on invalid strings
to_number("hello")      // Error: cannot convert

To Boolean:

to_boolean(1)           // true
to_boolean(0)           // false
to_boolean("text")      // true (non-empty string)
to_boolean("")          // false (empty string)
to_boolean(null)        // false
to_boolean([1])         // true (non-empty list)

// Explicit is often better
if len(text) > 0 {      // Better than: if to_boolean(text)
    // ...
}

Automatic Conversion:

Quill automatically converts types in some contexts:

// String concatenation converts to string
let text = "Count: " + 42           // "Count: 42"

// Conditions convert to boolean
if "text" {                         // Non-empty string is truthy
    // ...
}

// Comparison converts types
10 == "10"                          // false (no automatic conversion)

Best Practices:

  1. Be explicit when possible:
// Unclear
if value {
    // ...
}

// Clear
if value != null && len(value) > 0 {
    // ...
}
  1. Validate user input:
OnEvent(PlayerChat) {
    let input = chat.message

    try {
        let number = to_number(input)
        log("You entered: {number}")
    } catch error {
        sendmessage(player, "Please enter a valid number!")
    }
}
  1. Use type_of for safety:
func process_value(val) {
    if type_of(val) != "number" {
        log("Error: expected number")
        return
    }

    // Safe to use as number
    let result = val * 2
}
  1. Document expected types:
// Good: function comment documents types
// heal_player: Player -> Boolean
// Heals the given player to full health
// Returns true if successful
func heal_player(p) {
    if type_of(p) != "player" {
        return false
    }
    heal(p)
    return true
}
