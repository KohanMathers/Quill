Quill/Language/Syntax

Syntax

Syntax

Overview

Quill's syntax is designed to be clean, readable, and familiar to anyone who has used JavaScript, Python, or similar languages. This guide covers all the syntactic elements of the Quill language.

Key principles:
- Whitespace-agnostic - Indentation is for readability, not syntax
- Newline-terminated - Statements end at newlines (semicolons optional)
- C-style operators - Uses familiar +, -, ==, &&, etc.
- Curly-brace blocks - Code blocks use { } like JavaScript/Java

Comments

Comments are text that Quill ignores. Use them to explain your code, leave notes, or temporarily disable code.

Single-Line Comments

Single-line comments start with // and continue to the end of the line:

// This is a single-line comment
let x = 10  // Comments can also go at the end of lines

// You can use multiple single-line comments
// to create multi-line explanations
// like this

Multi-Line Comments

Multi-line comments start with /* and end with */:

/* This is a multi-line comment */

/* 
   You can span multiple lines
   to write longer explanations
   or documentation
*/

let x = 10  /* Inline comments work too */

Nesting is not supported:

/* This is a comment /* this is NOT a nested comment */ this breaks */

Use cases:
- Single-line for quick notes: // TODO: Optimize this
- Multi-line for documentation blocks or commenting out large sections

Variables

Variables store data that can be used and modified throughout your script.

Variable Declaration

Use let to declare a new variable:

let player_count        // Declared but undefined (value is null)
let score = 0           // Declared and initialized
let name = "Steve"      // Can be any data type

Rules:
- Variable names must start with a letter or underscore
- Can contain letters, numbers, and underscores
- Cannot use reserved keywords (let, if, func, etc.)
- Case-sensitive (score ≠ Score)

// Valid names
let player
let _internal
let player1
let player_score
let __private

// Invalid names
let 1player      // Cannot start with number
let player-name  // Cannot use hyphens
let if           // Cannot use keywords

Constants

Use const to declare values that should never change:

const MAX_PLAYERS = 10
const SPAWN_X = 0
const SPAWN_Y = 64
const SPAWN_Z = 0

Constants must be initialized when declared:

const MAX_PLAYERS        // Error: const must have value
const MAX_PLAYERS = 10   // Correct

Constants cannot be reassigned:

const MAX_PLAYERS = 10
MAX_PLAYERS = 20         // Error: cannot reassign const

Use constants for:
- Configuration values that don't change
- Magic numbers that need names
- Values computed once and reused

Variable Assignment

After declaration, assign new values with =:

let score = 0     // Initial value
score = 10        // Reassignment
score = score + 5 // Using current value

Compound assignment (not supported):

// These do NOT work in Quill
score += 5    // Error
score++       // Error

// Instead, write:
score = score + 5
score = score + 1

Multiple variables:

// Declare multiple variables separately
let x = 0
let y = 0
let z = 0

// Multiple declaration in one line (not supported)
let x = 0, y = 0, z = 0  // Error

Scope Rules

Variables have lexical scope - they're accessible where they're declared and in nested blocks:

let global_var = "I'm global"

func my_function() {
    let function_var = "I'm in the function"

    if true {
        let block_var = "I'm in the block"

        log(global_var)    // ✓ Accessible
        log(function_var)  // ✓ Accessible
        log(block_var)     // ✓ Accessible
    }

    log(global_var)    // ✓ Accessible
    log(function_var)  // ✓ Accessible
    log(block_var)     // ✗ Error: undefined
}

log(global_var)    // ✓ Accessible
log(function_var)  // ✗ Error: undefined

Variable shadowing:

let x = 10

func test() {
    let x = 20  // Different variable, shadows outer x
    log(x)      // Prints: 20
}

log(x)          // Prints: 10

Redeclaration is not allowed:

let x = 10
let x = 20  // Error: x already defined

Data Types Overview

Quill supports several data types. See Data Types for full details.

let num = 42                    // Number
let str = "Hello"               // String
let bool = true                 // Boolean
let nothing = null              // Null
let list = [1, 2, 3]           // List
let map = {name: "Steve"}      // Map

// Minecraft types
let player = get_player("Steve")         // Player
let loc = location(0, 64, 0)            // Location
let item = item("diamond_sword")         // Item

Operators

Arithmetic Operators

Perform mathematical operations:

let a = 10
let b = 3

let sum = a + b          // 13 (addition)
let diff = a - b         // 7  (subtraction)
let product = a * b      // 30 (multiplication)
let quotient = a / b     // 3.333... (division)
let remainder = a % b    // 1  (modulo)

Operator precedence (highest to lowest):
1. Parentheses ( )
2. Multiplication, Division, Modulo: * / %
3. Addition, Subtraction: + -

let result = 2 + 3 * 4      // 14 (not 20)
let result = (2 + 3) * 4    // 20 (parentheses first)

Negative numbers:

let neg = -5
let result = 10 + -5    // 5
let result = 10 - -5    // 15

Comparison Operators

Compare values and return true or false:

let a = 10
let b = 20

a == b    // false (equal to)
a != b    // true  (not equal to)
a > b     // false (greater than)
a < b     // true  (less than)
a >= b    // false (greater than or equal to)
a <= b    // true  (less than or equal to)

Type matters in equality:

10 == 10     // true
10 == 10.0   // true (same number)
10 == "10"   // false (number vs string)

Comparing strings:

"apple" == "apple"   // true
"apple" == "Apple"   // false (case-sensitive)

Logical Operators

Combine boolean expressions:

let x = 10
let y = 20

// AND - both must be true
x > 5 && y > 15     // true
x > 5 && y > 25     // false

// OR - at least one must be true
x > 5 || y > 25     // true
x < 5 || y > 25     // false

// NOT - inverts the value
!(x > 5)            // false
!(x < 5)            // true

Short-circuit evaluation:

// If first condition is false, second is never checked
false && expensive_function()  // expensive_function() NOT called

// If first condition is true, second is never checked
true || expensive_function()   // expensive_function() NOT called

Logical operator precedence:
1. NOT !
2. AND &&
3. OR ||

true || false && false    // true (AND first, then OR)
(true || false) && false  // false (OR first, then AND)

String Concatenation

Use + to join strings:

let first = "Hello"
let last = "World"
let greeting = first + " " + last  // "Hello World"

Automatic conversion:

let text = "Score: " + 100        // "Score: 100"
let text = "Player " + 1          // "Player 1"
let text = "Active: " + true      // "Active: true"

String interpolation is better:

let score = 100
let text = "Score: {score}"       // Preferred method

See String Interpolation for more details.

Control Flow

Control flow statements determine which code runs and when.

If Statements

Execute code conditionally:

if condition {
    // Code runs only if condition is true
}

Examples:

if player_count > 10 {
    broadcast("Server is crowded!")
}

if in_region(player, spawn) {
    heal(player)
}

if score == 100 {
    sendmessage(player, "Perfect score!")
}

Truthy and falsy values:

if 0 { }          // false
if 1 { }          // true
if "" { }         // false (empty string)
if "text" { }     // true
if null { }       // false
if [] { }         // false (empty list)
if [1] { }        // true

If-Else Statements

Execute different code based on condition:

if condition {
    // Runs if condition is true
} else {
    // Runs if condition is false
}

Examples:

if score > 100 {
    sendmessage(player, "High score!")
} else {
    sendmessage(player, "Keep trying!")
}

if get_gamemode(player) == "survival" {
    give(player, "apple")
} else {
    sendmessage(player, "Only survival players get apples")
}

Else-If Chains

Test multiple conditions:

if condition1 {
    // Runs if condition1 is true
} else if condition2 {
    // Runs if condition1 is false and condition2 is true
} else if condition3 {
    // Runs if condition1 and condition2 are false, condition3 is true
} else {
    // Runs if all conditions are false
}

Examples:

if score >= 90 {
    sendmessage(player, "Grade: A")
} else if score >= 80 {
    sendmessage(player, "Grade: B")
} else if score >= 70 {
    sendmessage(player, "Grade: C")
} else {
    sendmessage(player, "Grade: F")
}

First matching condition wins:

let x = 15

if x > 10 {
    log("Greater than 10")     // This runs
} else if x > 5 {
    log("Greater than 5")      // This is skipped
}

While Loops

Repeat code while a condition is true:

while condition {
    // Code runs repeatedly while condition is true
}

Examples:

let count = 0
while count < 5 {
    log("Count: {count}")
    count = count + 1
}
// Prints: 0, 1, 2, 3, 4

let health = 20
while health > 0 {
    health = health - 5
    log("Health: {health}")
}

Infinite loops (avoid!):

while true {
    // This never stops - will freeze your server!
    log("Forever")
}

Always ensure the condition eventually becomes false:

// Bad - count never changes
let count = 0
while count < 10 {
    log(count)  // Infinite loop!
}

// Good - count increases
let count = 0
while count < 10 {
    log(count)
    count = count + 1  // Eventually reaches 10
}

For Loops

Iterate over lists:

for variable in list {
    // Code runs once for each item in the list
}

Examples:

let players = get_online_players()
for player in players {
    heal(player)
}

let items = ["sword", "bow", "arrow"]
for item in items {
    log("Item: {item}")
}

// Range of numbers
for i in range(0, 5) {
    log("Number: {i}")  // 0, 1, 2, 3, 4
}

Iterating over scope players:

for p in my_scope.players {
    sendmessage(p, "Hello from the scope!")
}

Nested loops:

for x in range(0, 3) {
    for y in range(0, 3) {
        log("Position: {x}, {y}")
    }
}
// Prints all combinations: (0,0), (0,1), (0,2), (1,0), ...

Break and Continue

Control loop execution:

Break - Exit the loop immediately:

for i in range(0, 10) {
    if i == 5 {
        break  // Stop the loop when i is 5
    }
    log(i)  // Prints: 0, 1, 2, 3, 4
}

Continue - Skip to the next iteration:

for i in range(0, 10) {
    if i % 2 == 0 {
        continue  // Skip even numbers
    }
    log(i)  // Prints: 1, 3, 5, 7, 9
}

Examples:

// Find first player with low health
for player in get_online_players() {
    if get_health(player) < 5 {
        heal(player)
        break  // Stop after healing first low-health player
    }
}

// Heal all players except spectators
for player in get_online_players() {
    if get_gamemode(player) == "spectator" {
        continue  // Skip spectators
    }
    heal(player)
}

Break/continue only affect innermost loop:

for x in range(0, 3) {
    for y in range(0, 3) {
        if y == 1 {
            break  // Only breaks inner loop
        }
        log("{x}, {y}")
    }
}

Expressions

Expressions are code that produces a value.

Binary Expressions

Expressions with two operands and an operator:

10 + 5          // Arithmetic
x > 10          // Comparison
x && y          // Logical
"Hello" + " "   // String concatenation

// Can be nested
(10 + 5) * 2           // 30
x > 10 && y < 20       // Compound condition

Unary Expressions

Expressions with one operand and an operator:

-10             // Negation
!true           // Logical NOT
-(5 + 3)        // Negate result

// Examples
let x = 10
let neg_x = -x              // -10
let opposite = !is_active   // Invert boolean

Parentheses

Use parentheses to:
1. Control precedence:

2 + 3 * 4       // 14 (multiplication first)
(2 + 3) * 4     // 20 (addition first)
  1. Group logical conditions:
if (x > 10 && x < 20) || x == 0 {
    // ...
}
  1. Improve readability:
// Less clear
if x > 10 && y > 10 || z > 10 { }

// More clear
if (x > 10 && y > 10) || z > 10 { }

Statements vs Expressions

Understanding the difference helps write better code.

Expressions produce values:

10 + 5                    // Produces: 15
x > 10                    // Produces: true or false
get_health(player)        // Produces: a number

Statements perform actions:

let x = 10               // Declares variable (no value produced)
if x > 5 { }            // Executes conditionally
for i in range(0, 10) { } // Loops

Expression statements - Expressions used as statements:

heal(player)             // Function call used as statement
player_count + 1         // Valid but useless (result discarded)

Expressions can be used in statements:

let score = player_count * 10    // Expression assigned to variable
if get_health(player) < 10 { }   // Expression used as condition

Statements cannot be used as expressions:

let x = if true { 10 }           // Error: if is not an expression
let y = while false { 0 }        // Error: while is not an expression

Semicolons

Semicolons are optional in Quill. Statements are terminated by newlines.

Both are valid:

// Without semicolons (recommended)
let x = 10
let y = 20
let z = x + y

// With semicolons
let x = 10;
let y = 20;
let z = x + y;

Multiple statements on one line require semicolons:

let x = 10; let y = 20; let z = 30  // Multiple statements

However, this style is discouraged:

// Bad - hard to read
let x = 10; let y = 20; if x > y { log("x wins") }

// Good - each statement on its own line
let x = 10
let y = 20
if x > y {
    log("x wins")
}

Semicolons are never harmful:

// If you're used to languages that require semicolons,
// feel free to keep using them
let x = 10;
log(x);

Best Practices

1. Use Meaningful Names

// Bad
let x = 10
let p = get_player("Steve")

// Good
let player_count = 10
let target_player = get_player("Steve")

2. Consistent Naming Convention

// Use snake_case for variables and functions
let player_count = 0
func heal_all_players() { }

// Use UPPER_SNAKE_CASE for constants
const MAX_PLAYERS = 10
const SPAWN_LOCATION = location(0, 64, 0)

3. Use Constants for Magic Numbers

// Bad
if score > 1000 {
    give(player, "diamond", 10)
}

// Good
const HIGH_SCORE_THRESHOLD = 1000
const REWARD_AMOUNT = 10

if score > HIGH_SCORE_THRESHOLD {
    give(player, "diamond", REWARD_AMOUNT)
}

4. Keep Conditions Simple

// Bad - hard to read
if player_count > 10 && game_active && !is_paused && has_permission { }

// Good - extract to variables
let has_enough_players = player_count > 10
let can_start_game = game_active && !is_paused && has_permission

if has_enough_players && can_start_game { }

5. Use Early Returns

// Bad - nested conditions
func process_player(player) {
    if is_online(player) {
        if get_gamemode(player) == "survival" {
            if get_health(player) < 10 {
                heal(player)
            }
        }
    }
}

// Good - early returns
func process_player(player) {
    if !is_online(player) {
        return
    }

    if get_gamemode(player) != "survival" {
        return
    }

    if get_health(player) >= 10 {
        return
    }

    heal(player)
}

6. Comment Why, Not What

// Bad - obvious from code
let x = x + 1  // Add 1 to x

// Good - explains reasoning
let x = x + 1  // Offset by 1 for array indexing

// Bad
if score > 100 {  // Check if score is greater than 100
    give_reward(player)
}

// Good
if score > 100 {  // Veteran players deserve extra rewards
    give_reward(player)
}

7. Consistent Formatting

// Good - consistent spacing
let x = 10
let y = 20
let result = x + y

// Bad - inconsistent spacing
let x=10
let y = 20
let result =x+y

8. Use Whitespace for Readability

// Good - logical grouping
const MAX_PLAYERS = 10
const MAX_SCORE = 1000

let current_players = 0
let current_score = 0

func start_game() {
    // ...
}

func end_game() {
    // ...
}

9. Limit Line Length

// Bad - too long
if player_count > 10 && game_active && !is_paused && has_permission && is_weekend && temperature > 20 { }

// Good - break into multiple lines
if player_count > 10 && 
   game_active && 
   !is_paused && 
   has_permission {
    // ...
}

// Better - extract to variables
let has_enough_players = player_count > 10
let game_is_ready = game_active && !is_paused && has_permission

if has_enough_players && game_is_ready {
    // ...
}

10. Avoid Deep Nesting

// Bad - 4 levels deep
func process() {
    if a {
        if b {
            if c {
                if d {
                    do_something()
                }
            }
        }
    }
}

// Good - use early returns
func process() {
    if !a { return }
    if !b { return }
    if !c { return }
    if !d { return }

    do_something()
}
