Language — Syntax

Language — Syntax

This page covers the core Quill syntax you need to write scripts: comments, variables, types, operators, control flow, and functions. It is a concise reference derived from the language specification.

Comments

// Single-line comment
/* Multi-line
   comment */

Use comments to explain intent or temporarily disable code.

Variables

Declare variables with let (dynamically typed):

let count = 0;
let name = "Player";

Root-level let variables are global to the scope. Assigning to properties of a Scope creates subscope-local variables.

Data types

Primitives
- Number: 42, 3.14, -7
- String: "hello", 'world'
- Boolean: true, false
- Null: null

Complex
- Player — a reference to an online player
- Location — location(x, y, z)
- Item — item("minecraft:diamond_sword", amount, metadata)
- Scope — new Scope(x1, y1, z1, x2, y2, z2)
- List — [1, 2, "a"]
- Entity — game entity reference

Use type_of(value) to inspect types at runtime.

Operators

Arithmetic

+  -  *  /  %

Comparison

==  !=  >  <  >=  <=

Logical

&&  ||  !

String

"a" + "b"        // concatenation
"Hello {name}"    // interpolation using { }

Control flow

If statements

if condition {
    // code
} else if other {
    // code
} else {
    // code
}

For loops

for item in list {
    // iterate list
}

for i in range(0, 10) {
    // iterate numeric range
}

for p in subscope.players {
    // enters the subscope context
}

While loops

while condition {
    // code
}

Loop control
- break — exit loop
- continue — next iteration

Functions

Define functions with func:

func add(a, b) {
    return a + b;
}

func hello(name) {
    log(name, "Hi!");
}

Call functions like result = add(1, 2) or hello(player).

Scope basics (short)

Create subscopes with new Scope(x1, y1, z1, x2, y2, z2). Scopes provide:
- scope.players — list of players in that scope
- in_region(player, scope) — check membership

Context rules
- Root-level variables are always accessible.
- Subscope-local variables are only available when executing within that subscope's context (for example, when iterating subscope.players).

Quick examples

Variables and interpolation

let greeting = "Welcome"
sendmessage(player, "{greeting}, {player.name}!")

Simple function and loop

func give_starter(player) {
    give(player, "minecraft:bread", 5);
    sendmessage(player, "Starter kit given.");
}

for p in get_online_players() {
    give_starter(p);
}
