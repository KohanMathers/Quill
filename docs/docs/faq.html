Frequently Asked Questions

Frequently Asked Questions

General Questions

What is Quill?

Quill allows players to write custom scripts that react to in-game events within defined physical boundaries. Scripts run in isolated scopes with granular permission controls, making it safe for server owners to delegate creativity to their community.

Who should use Quill?

Quill is designed for server owners that want to delegate creativity to their members without giving them the full freedom other scripting plugins provide.

Is Quill safe for production servers?

The core language and API are stable, but minor breaking changes may still occur.

What versions of Minecraft does Quill support?

Quill is developed and tested on 1.21, but may work on older versions although untested.

Scripting Questions

What language does Quill use?

Quill's syntax is designed to be clean, readable, and familiar to anyone who has used JavaScript, Python, or similar languages.

Can I use JavaScript/Python/etc?

No, although Quill's syntax is similar it is still unique.

How do I learn Quill?

As of writing (15/11/2025) the only available tutorial is the guides in this documentation.

Where do I put my scripts?

plugins/Quill/scripts

Scope Questions

What is a scope?

A scope is a context that defines:
1. Variable accessibility - Which variables can be read and written
2. Physical boundaries - A 3D region in the Minecraft world (optional)
3. Player membership - Which players belong to the scope
4. Function permissions - Which built-in functions can be called
5. Persistent variables - Variables that survive script reloads

Do I need to create a scope?

Not necessarily. Although you can run all scopes in the global scope, you should create scopes for security.

Can scopes overlap?

Yes, and players will be marked as being in both scope's regions.

How many scopes can I have?

As many as you want, although for performance reasons the recommended max is ~20

Security Questions

Can scripts harm my server?

If they are not properly scoped, there is a low chance a malicious script could overwork the server to the point of harm.

What permissions should I give players?

The general recommendation for permissions for a normal player are as follows

quill.command
quill.load.scope
quill.reload.scope
quill.unload.scope
quill.list
quill.info
quill.help
quill.scope.list
quill.scope.info
quill.scope.persist

How do I restrict dangerous functions?

  • /quill scope mode <name> blacklist
  • /quill scope permission revoke <permission>
    And repeat for all permissions you wish to restrict

Can scripts access files?

No, although that is planned for a future release.

Performance Questions

Will Quill slow down my server?

On average, Quill runs extremely performant unless you have 20 or more scopes running 3 or more scripts each.

How many scripts can I run?

Unlimited, but refer to the question above for the recommended amount

Do events impact TPS?

Refer to Will Quill slow down my server?

Should I optimize my scripts?

Yes.

Integration Questions

Can Quill work with other plugins?

Quill can work alongside other plugins, although it has no integration with them.

Can I trigger Quill from other plugins?

Not currently, although you can modify events before they reach Quill (see API Integration)
