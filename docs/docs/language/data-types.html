Language — Data Types

Quill supports a range of primitive and complex data types for scripting Minecraft events and logic. This page summarizes all available types, their properties, and how to inspect them at runtime.

Primitives

  • Number: 42, 3.14, -100
  • String: "hello", 'world'
  • Boolean: true, false
  • Null: null

Numbers are always floating-point. Strings can use single or double quotes.

Complex types

  • Player: Reference to a Minecraft player (online or offline)
  • Location: location(x, y, z) or location(x, y, z, world)
  • Item: item("minecraft:diamond_sword", amount, metadata)
  • Scope: Reference to a subscope (region)
  • List: [1, 2, 3, "hello", true] — dynamic array of any type
  • Entity: Reference to a Minecraft entity (mob, dropped item, etc.)
  • World: Reference to a Minecraft world
  • Region: Object with x1, y1, z1, x2, y2, z2 (scope bounds)

Object properties

Player

  • player.name — String
  • player.health — Number
  • player.hunger — Number
  • player.location — Location
  • player.gamemode — String
  • player.flying — Boolean
  • player.online — Boolean

Location

  • location.x, location.y, location.z — Number
  • location.world — World

Item

  • item.type — String (e.g., "minecraft:diamond_sword")
  • item.amount — Number
  • item.metadata — Object (NBT data)

Entity

  • entity.type — String
  • entity.location — Location
  • entity.health — Number
  • entity.alive — Boolean

Scope

  • scope.players — List of players in the scope
  • scope.region — Region object

Region

  • region.x1, region.y1, region.z1, region.x2, region.y2, region.z2 — Number ⚠️ Not yet implemented

Lists

Lists are dynamic arrays and can hold any type:

let items = [1, "a", true, player]
let players = scope.players

Use len(list) to get the length, append(list, item) to add, remove(list, index) to remove, and contains(list, item) to check for membership.

Type inspection

Use type_of(value) to get the type name as a string:

let t = type_of(player)  // "player"
let t2 = type_of([1,2])  // "list"

Type conversion

Convert between types with:
- to_string(value)
- to_number(value)
- to_boolean(value)

Null and existence

null is used for missing values, failed lookups, or uninitialized variables. Always check for null before accessing properties.
