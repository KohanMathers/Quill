String Interpolation

String Interpolation

Overview

String interpolation in Quill allows you to embed variable values and expressions directly within string literals using curly brace syntax {...}. At runtime, the expressions are evaluated and converted to strings, making it easy to create dynamic messages without manual concatenation.

String interpolation works in any string literal (single or double quotes) and supports variables, member access, and nested properties. This feature is particularly useful for player messages, logging, and displaying dynamic game state.

Basic Syntax

Simple Variables

Wrap variable names in curly braces to interpolate them:

let playerName = "Steve"
let message = "Hello, {playerName}!"
// Result: "Hello, Steve!"

The variable is evaluated and converted to a string at the point where the string is created.

Examples

With numbers:

let score = 1500
let text = "Your score: {score}"
// Result: "Your score: 1500"

With booleans:

let isOnline = true
let status = "Online status: {isOnline}"
// Result: "Online status: true"

In event handlers:

OnEvent(PlayerJoin) {
    let greeting = "Welcome, {player.name}!"
    sendmessage(player, greeting)
    // Result: "Welcome, Steve!"
}

In function calls:

OnEvent(BlockBreak) {
    log("Player {player.name} broke {block.type}")
    // Result: "Player Steve broke diamond_ore"
}

Expressions in Interpolation

Member Access

Access object properties using dot notation:

OnEvent(PlayerJoin) {
    sendmessage(player, "Welcome, {player.name}!")
    // Accesses the 'name' property of player
}

OnEvent(PlayerMove) {
    let x = move.to.x
    let y = move.to.y
    let z = move.to.z
    sendmessage(player, "Position: {move.to.x}, {move.to.y}, {move.to.z}")
    // Result: "Position: 123.5, 64.0, -45.2"
}

Nested Properties

Chain multiple property accesses:

OnEvent(PlayerMove) {
    log("World: {move.to.world.name}")
    // Accesses: move -> to -> world -> name
}

OnEvent(BlockBreak) {
    sendmessage(player, "Broke {block.type} at X: {block.location.x}")
    // Accesses nested location properties
}

Available nested properties:

Player properties:
- {player.name} - Username
- {player.health} - Current health
- {player.hunger} - Current hunger
- {player.location.x} - X coordinate
- {player.location.y} - Y coordinate
- {player.location.z} - Z coordinate
- {player.gamemode} - Current gamemode
- {player.flying} - Flight status
- {player.online} - Online status

Location properties:
- {location.x} - X coordinate
- {location.y} - Y coordinate
- {location.z} - Z coordinate
- {location.world.name} - World name

Item properties:
- {item.type} - Item material
- {item.amount} - Stack size

Entity properties:
- {entity.type} - Entity type
- {entity.location.x} - X coordinate
- {entity.alive} - Alive status

Limitations

Function calls are NOT supported:

// DOES NOT WORK
let message = "Health: {get_health(player)}"
// Error: Function calls not supported in interpolation

Arithmetic expressions are NOT supported:

// DOES NOT WORK
let x = 10
let text = "Result: {x + 5}"
// Error: Arithmetic not supported in interpolation

Complex expressions are NOT supported:

// DOES NOT WORK
let text = "Status: {player.health > 10}"
// Error: Comparisons not supported

Workaround - Calculate first, then interpolate:

// CORRECT APPROACH
let health = get_health(player)
let message = "Health: {health}"

let x = 10
let result = x + 5
let text = "Result: {result}"

let isHealthy = player.health > 10
let status = "Healthy: {isHealthy}"

Advanced Usage

Multiple Variables

Use multiple interpolations in a single string:

OnEvent(PlayerDamage) {
    let attacker = damage.source
    let victim = damage.target
    let amount = damage.amount

    log("{victim.name} took {amount} damage from {attacker.type}")
    // Result: "Steve took 5.0 damage from zombie"
}

OnEvent(BlockPlace) {
    let x = block.location.x
    let y = block.location.y
    let z = block.location.z
    let type = block.type

    sendmessage(player, "Placed {type} at {x}, {y}, {z}")
    // Result: "Placed stone at 100.0, 64.0, 200.0"
}

Formatting

Concatenate with static text:

OnEvent(PlayerJoin) {
    sendmessage(player, "=== Welcome {player.name}! ===")
    sendmessage(player, "Current health: {player.health}/20")
    sendmessage(player, "Position: ({player.location.x}, {player.location.y}, {player.location.z})")
}

Build complex messages:

func displayPlayerInfo(player) {
    let info = "Player: {player.name}\nHealth: {player.health}\nHunger: {player.hunger}\nGamemode: {player.gamemode}"
    sendmessage(player, info)
}

Create structured output:

OnEvent(BlockBreak) {
    let report = "BLOCK BROKEN\nPlayer: {player.name}\nBlock: {block.type}\nLocation: {block.location.x}, {block.location.y}, {block.location.z}"
    log(report)
}

Common Use Cases

Player Names

Welcome messages:

OnEvent(PlayerJoin) {
    sendmessage(player, "Welcome back, {player.name}!")
    broadcast("{player.name} has joined the server!")
}

Action notifications:

OnEvent(PlayerDeath) {
    if death.killer != null {
        broadcast("{death.killer.name} defeated {player.name}!")
    }
}

Coordinates

Location display:

OnEvent(PlayerChat) {
    if chat.message == "!pos" {
        let x = player.location.x
        let y = player.location.y
        let z = player.location.z
        sendmessage(player, "Position: {x}, {y}, {z}")
    }
}

Distance calculations:

let spawn = location(0, 64, 0)

OnEvent(PlayerChat) {
    if chat.message == "!distance" {
        let dist = distance(player.location, spawn)
        sendmessage(player, "You are {dist} blocks from spawn")
    }
}

Region boundaries:

func showRegion(scope) {
    let region = get_region(scope)
    log("Region from ({region.x1}, {region.y1}, {region.z1}) to ({region.x2}, {region.y2}, {region.z2})")
}

Status Messages

Health and hunger:

OnEvent(PlayerChat) {
    if chat.message == "!stats" {
        sendmessage(player, "Health: {player.health}/20")
        sendmessage(player, "Hunger: {player.hunger}/20")
        sendmessage(player, "Gamemode: {player.gamemode}")
    }
}

Event logging:

OnEvent(BlockBreak) {
    log("{player.name} broke {block.type} at {block.location.x}, {block.location.y}, {block.location.z}")
}

OnEvent(EntitySpawn) {
    log("Entity spawned: {entity.type} at {entity.location.x}, {entity.location.y}, {entity.location.z}")
}

Game state:

let gameActive = false
let playerCount = 0

OnEvent(PlayerChat) {
    if chat.message == "!game" {
        sendmessage(player, "Game active: {gameActive}")
        sendmessage(player, "Players: {playerCount}")
    }
}

Gotchas and Limitations

What Works

✓ Simple variables:

let name = "Steve"
log("Name: {name}")  // ✓ Works

✓ Object properties:

log("Player: {player.name}")  // ✓ Works

✓ Nested properties:

log("X: {player.location.x}")  // ✓ Works

✓ Multiple interpolations:

log("{player.name} at {player.location.x}, {player.location.y}")  // ✓ Works

✓ All basic types:

let num = 42
let bool = true
let str = "hello"
log("Num: {num}, Bool: {bool}, Str: {str}")  // ✓ Works

✓ Map properties:

let data = { name: "Steve", age: 25 }
log("Name: {data.name}")  // ✓ Works

What Doesn't Work

✗ Function calls:

log("Health: {get_health(player)}")  // ✗ Doesn't work
// Solution: Calculate first
let health = get_health(player)
log("Health: {health}")  // ✓ Works

✗ Arithmetic operations:

let x = 10
log("Double: {x * 2}")  // ✗ Doesn't work
// Solution: Calculate first
let doubled = x * 2
log("Double: {doubled}")  // ✓ Works

✗ Comparisons:

log("Healthy: {player.health > 10}")  // ✗ Doesn't work
// Solution: Evaluate first
let isHealthy = player.health > 10
log("Healthy: {isHealthy}")  // ✓ Works

✗ Logical operations:

log("Valid: {x > 0 && x < 100}")  // ✗ Doesn't work
// Solution: Evaluate first
let isValid = x > 0 && x < 100
log("Valid: {isValid}")  // ✓ Works

✗ Array/list indexing:

let players = get_online_players()
log("First: {players[0]}")  // ✗ Doesn't work
// Solution: Extract first
let first = players[0]
log("First: {first}")  // ✗ Still doesn't work - use first.name

✗ Ternary operators:

log("Status: {player.health > 10 ? 'Healthy' : 'Hurt'}")  // ✗ Doesn't work
// Note: Quill doesn't support ternary operators at all
// Solution: Use if/else
let status = "Hurt"
if player.health > 10 {
    status = "Healthy"
}
log("Status: {status}")  // ✓ Works

Best Practices

Calculate before interpolating:

// Good
let distance = distance(player.location, spawn)
let rounded = round(distance)
sendmessage(player, "Distance: {rounded} blocks")

// Avoid
sendmessage(player, "Distance: {distance(player.location, spawn)}")  // Won't work

Keep interpolations simple:

// Good - clear and readable
let name = player.name
let health = player.health
sendmessage(player, "{name} has {health} health")

// Avoid - too complex
sendmessage(player, "{player.name} has {player.health} health remaining out of 20 maximum")

Use meaningful variable names:

// Good
let playerName = player.name
let currentHealth = player.health
log("{playerName} health: {currentHealth}")

// Avoid
let p = player.name
let h = player.health
log("{p} health: {h}")

Format consistently:

// Good - consistent formatting
sendmessage(player, "X: {x}, Y: {y}, Z: {z}")
sendmessage(player, "Health: {health}/20")
sendmessage(player, "Score: {score} points")

// Avoid - inconsistent formatting
sendmessage(player, "X:{x} Y:{y} Z:{z}")
sendmessage(player, "Health:{health} / 20")
sendmessage(player, "{score}points")

Validate data exists before interpolating:

// Good - check for null
OnEvent(PlayerDeath) {
    if death.killer != null {
        broadcast("{death.killer.name} defeated {player.name}!")
    } else {
        broadcast("{player.name} died!")
    }
}

// Avoid - may error if killer is null
OnEvent(PlayerDeath) {
    broadcast("{death.killer.name} defeated {player.name}!")  // Crashes if no killer
}

Use temporary variables for readability:

// Good - readable
let attacker = damage.source.type
let victim = damage.target.name
let amount = damage.amount
log("{victim} took {amount} damage from {attacker}")

// Avoid - hard to read
log("{damage.target.name} took {damage.amount} damage from {damage.source.type}")

Combine with string functions when needed:

let name = player.name
let uppercase = to_string(name)  // Convert if needed
sendmessage(player, "Hello {uppercase}!")

Document what's being interpolated:

// Sends a formatted position message
// Format: "Position: X, Y, Z"
func sendPosition(player) {
    let x = player.location.x
    let y = player.location.y
    let z = player.location.z
    sendmessage(player, "Position: {x}, {y}, {z}")
}
