Functions

Functions

Overview

Functions in Quill are reusable blocks of code that can accept parameters and return values. Quill supports both user-defined functions and a comprehensive set of built-in functions for interacting with the Minecraft world.

Declaring Functions

Basic Function Declaration

Functions are declared using the func or function keyword:

func greet() {
    log("Hello, world!")
}

Function Parameters

Functions can accept parameters that become local variables within the function body:

func greet(name) {
    log("Hello, {name}!")
}

func add(a, b) {
    return a + b
}

Important: All parameters must be provided when calling a function. The number of arguments must match the number of parameters exactly.

Return Values

Functions can return values using the return statement:

func multiply(x, y) {
    return x * y
}

let result = multiply(5, 3)  // result = 15

If no return statement is used, functions return null by default.

Calling Functions

Basic Function Calls

Call functions by using their name followed by parentheses:

greet()  // Calls function with no parameters
greet("Alice")  // Calls function with one parameter

Passing Arguments

Arguments are passed by value and evaluated before the function executes:

func calculate(x, y) {
    return x * 2 + y
}

let a = 5
let b = 3
let result = calculate(a + 1, b * 2)  // calculate(6, 6)

Using Return Values

Capture return values by assigning the function call to a variable:

func getPlayerHealth(player) {
    return get_health(player)
}

let health = getPlayerHealth(player)
sendmessage(player, "Your health: {health}")

Function Scope

Local Variables

Variables declared inside functions are local to that function:

func example() {
    let localVar = 10  // Only exists inside this function
    log(localVar)
}

example()
// log(localVar)  // Error: undefined variable

Closure

Functions capture their surrounding scope and can access variables from parent scopes:

let multiplier = 5

func multiply(x) {
    return x * multiplier  // Accesses outer scope
}

log(multiply(3))  // Output: 15

Accessing Outer Scope

Functions can read and modify variables from outer scopes:

let counter = 0

func increment() {
    counter = counter + 1
}

increment()
increment()
log(counter)  // Output: 2

Built-in Functions

Quill provides extensive built-in functions organized by category:

Player Functions

Movement & Teleportation
- teleport(player, location) - Teleport player to location
- teleport(player, x, y, z) - Teleport player to coordinates

Inventory Management
- give(player, item_id) - Give item to player
- give(player, item_id, amount) - Give specific amount
- give(player, item) - Give item stack
- remove_item(player, item_id) - Remove item from inventory
- remove_item(player, item_id, amount) - Remove specific amount
- has_item(player, item_id) - Check if player has item
- has_item(player, item_id, amount) - Check for specific amount

Player State
- set_gamemode(player, mode) - Set gamemode (survival, creative, adventure, spectator)
- set_health(player, health) - Set health (0-20)
- set_hunger(player, hunger) - Set hunger (0-20)
- heal(player) - Fully heal player
- kill(player) - Kill player
- set_flying(player, flying) - Enable/disable flight
- kick(player, reason) - Kick player from server

Communication
- sendmessage(player, message) - Send chat message
- sendtitle(player, title, subtitle, fade_in, stay, fade_out) - Display title
- playsound(player, sound, volume, pitch) - Play sound

Effects
- give_effect(player, effect, duration, amplifier) - Apply potion effect
- remove_effect(player, effect) - Remove potion effect
- clear_effects(player) - Remove all effects

Getters
- get_health(player) - Get current health
- get_hunger(player) - Get current hunger
- get_location(player) - Get player location
- get_gamemode(player) - Get current gamemode
- get_name(player) - Get player name
- is_online(player) - Check if player is online
- is_op(player) - Check if player is operator

World Functions

Block Manipulation
- set_block(location, block_id) - Set block at location
- set_block(x, y, z, block_id) - Set block at coordinates
- get_block(location) - Get block type at location
- get_block(x, y, z) - Get block type at coordinates
- break_block(location) - Break block and drop items
- break_block(x, y, z) - Break block at coordinates

Entity Management
- spawn_entity(entity_type, location) - Spawn entity
- spawn_entity(entity_type, x, y, z) - Spawn at coordinates
- remove_entity(entity) - Remove entity from world

World Effects
- create_explosion(location, power) - Create explosion
- create_explosion(location, power, fire) - Explosion with fire
- create_explosion(x, y, z, power) - Explosion at coordinates
- create_explosion(x, y, z, power, fire) - Full control
- strike_lightning(location) - Strike lightning
- strike_lightning(x, y, z) - Lightning at coordinates

Time & Weather
- set_time(world, time) - Set world time (0-24000)
- get_time(world) - Get current time
- set_weather(world, weather, duration) - Set weather (clear, rain, thunder)
- get_weather(world) - Get current weather

Utilities
- get_world(name) - Get world by name
- broadcast(message) - Broadcast to all players in scope

Scope Functions

  • addtoscope(player, scope) - Add player to scope
  • removefromscope(player, scope) - Remove player from scope
  • getplayers(scope) - Get all players in scope
  • in_region(player, scope) - Check if player is in region
  • in_region(location, scope) - Check if location is in region
  • get_region(scope) - Get scope's region boundaries
  • set_region(scope, x1, y1, z1, x2, y2, z2) - Set region boundaries

Utility Functions

Math
- random(max) - Random number 0 to max
- random(min, max) - Random number between min and max
- round(number) - Round to nearest integer
- floor(number) - Round down
- ceil(number) - Round up
- abs(number) - Absolute value
- sqrt(number) - Square root
- pow(number, exponent) - Power
- distance(location1, location2) - Calculate distance
- distance(x1, y1, z1, x2, y2, z2) - Distance between coordinates

Type Conversion
- to_string(value) - Convert to string
- to_number(value) - Convert to number
- to_boolean(value) - Convert to boolean
- type_of(value) - Get value type

String Operations
- split(string, delimiter) - Split string into list
- join(list, delimiter) - Join list into string
- contains(string, substring) - Check if string contains substring
- len(string) - Get string length

List Operations
- len(list) - Get list length
- append(list, item) - Add item to list
- remove(list, index) - Remove item at index
- contains(list, item) - Check if list contains item
- range(start, end) - Create range of numbers
- random_choice(list) - Pick random item from list

Other
- log(message) - Log message to console
- cancel(event) - Cancel Bukkit event
- wait(ticks) - Wait specified ticks (non-blocking)
- trigger_custom(event_name, data) - Trigger custom event
- get_player(name) - Get player by name
- get_online_players() - Get all online players

Constructor Functions

  • location(x, y, z) - Create location in current world
  • location(x, y, z, world) - Create location in specific world
  • item(item_id) - Create item stack
  • item(item_id, amount) - Create item with amount
  • item(item_id, amount, metadata) - Create item with metadata

Arrow Functions (Not Supported)

Quill does not support arrow function syntax (=>). All functions must be declared using the func or function keyword.

Best Practices

Function Naming

Use descriptive, verb-based names for functions:

// Good
func healPlayer(player) { ... }
func checkInventorySpace(player) { ... }

// Avoid
func hp(p) { ... }
func check(x) { ... }

Parameter Count

Keep parameter count reasonable (ideally 3 or fewer). For more complex data, use maps:

// Good
func configurePlayer(config) {
    set_health(config.player, config.health)
    set_gamemode(config.player, config.gamemode)
}

// Less ideal
func configurePlayer(player, health, hunger, gamemode, effects) { ... }

Single Responsibility

Each function should do one thing well:

// Good - separate concerns
func isPlayerHealthy(player) {
    return get_health(player) >= 15
}

func healIfNeeded(player) {
    if !isPlayerHealthy(player) {
        heal(player)
    }
}

// Avoid - mixed concerns
func checkAndHealPlayer(player) {
    if get_health(player) < 15 {
        heal(player)
        sendmessage(player, "Healed!")
        log("Player healed")
    }
}

Documentation

Document complex functions with comments:

// Calculates damage reduction based on armor and enchantments
// Returns: damage multiplier (0.0 to 1.0)
func calculateDamageReduction(armor, enchantLevel) {
    let baseReduction = armor * 0.04
    let enchantBonus = enchantLevel * 0.03
    return 1 - (baseReduction + enchantBonus)
}
