General Commands

General Commands

Overview

Quill provides general utility commands for viewing plugin information, accessing help documentation, and managing your interaction with the plugin. These commands are foundational for understanding and using Quill effectively.

/quill info

Display information about the Quill plugin, including version, loaded scripts, and directory paths.

Syntax

/quill info

Permissions

  • quill.info (default: true)

Output

The command displays:
- Version: The current Quill plugin version
- Scripts Loaded: Number of currently active scripts
- Scripts Directory: Absolute filesystem path to the scripts folder

Example

/quill info

Sample Output:

=== Quill Plugin Info ===
Version: 0.2.0
Scripts Loaded: 5
Scripts Directory: /server/plugins/Quill/scripts
========================

Use Cases:
- Verify plugin version for support requests
- Check how many scripts are currently active
- Confirm the scripts directory path for manual file operations
- Troubleshooting script loading issues

/quill help

Display help information for Quill commands. Can show general help or topic-specific help.

Syntax

/quill help [topic]

Parameters

  • [topic] - Optional. The help topic to display
  • Omit for general command overview
  • Use scope for scope-specific commands

Permissions

  • quill.help (default: true)

Topics

General Help (no topic)

Shows all main Quill commands with brief descriptions.

Command:

/quill help

Output:

=== Quill Commands ===
/quill load <filename> - Load a script
/quill reload <filename> - Reload a script
/quill unload <filename> - Unload a script
/quill edit <filename> - Edit a script
/quill list - List all scripts
/quill info - Show plugin info
/quill scope <subcommand> [args] - Manage scopes (use /quill help scope)
/quill help - Show this help
=====================

Scope Help

Shows detailed information about scope management commands.

Command:

/quill help scope

Output:

=== Scope Commands ===
/quill scope create <name> <owner> <x1> <y1> <z1> <x2> <y2> <z2> <whitelist|blacklist> - Create a scope
/quill scope delete <name> - Delete a scope
/quill scope list - List all scripts
/quill scope info <name> - Show info about a scope
/quill scope permission <grant|revoke> <name> <function> - Grant or revoke a scope function
/quill scope persist <name> <variable> <add|remove> - Add or remove a persistent variable from scope
=====================

Examples

View general help:

/quill help

View scope-specific help:

/quill help scope

Quick reference for new users:

/quill help
/quill list
/quill info

Command Aliases

Quill provides command aliases to reduce typing and improve command-line efficiency.

Available Aliases

Full Command Alias Description
/quill /ql Main command shortcut

Usage Examples

Standard syntax:

/quill load welcome.ql
/quill reload myscript
/quill scope list

Using alias:

/ql load welcome.ql
/ql reload myscript
/ql scope list

All subcommands work with the alias:

/ql info
/ql help
/ql list
/ql edit welcome.ql
/ql scope create myScope Player 0 64 0 100 120 100 whitelist

Tab Completion

Quill provides intelligent tab completion for all commands, making it easier to discover and use commands correctly.

First Argument (Subcommands)

Press Tab after /quill to see all available subcommands:

/quill <Tab>

Completions:
- load
- reload
- unload
- edit
- list
- info
- scope
- help

Script Names

Press Tab after load/reload/unload/edit commands to see available scripts:

/quill load <Tab>
/quill reload <Tab>
/quill unload <Tab>
/quill edit <Tab>

Completions:
- All .ql and .quill files in the scripts directory
- Includes scope scripts (e.g., myScope/events.ql)
- Filtered based on your permissions

Example:

/quill load w<Tab>

Completes to:

/quill load welcome.ql

Scope Subcommands

Press Tab after /quill scope to see scope-specific commands:

/quill scope <Tab>

Completions:
- create
- delete
- list
- info
- permission
- persist
- mode

Scope Names

Press Tab when a scope name is expected:

/quill scope info <Tab>
/quill scope delete <Tab>
/quill scope permission grant <Tab>

Completions:
- All loaded scope names
- Filtered based on your permissions (owners see their scopes, ops see all)

Permission Actions

Press Tab for permission subcommands:

/quill scope permission <Tab>

Completions:
- grant
- revoke
- list

Persist Actions

Press Tab for persist subcommands:

/quill scope persist <Tab>

Completions:
- add
- remove
- list

Security Modes

Press Tab when setting scope mode:

/quill scope mode myScope <Tab>

Completions:
- whitelist
- blacklist

Smart Completion Examples

Loading a scope script:

/quill load my<Tab>

Might complete to:

/quill load myScope/<Tab>

Then:

/quill load myScope/events.ql

Managing scope permissions:

/quill scope per<Tab>

Completes to:

/quill scope permission <Tab>

Then:

/quill scope permission grant <Tab>

Shows all scope names:

/quill scope permission grant myScope

Permission Hierarchy

Quill uses a hierarchical permission system that allows fine-grained control over who can do what. Understanding this hierarchy is crucial for server administrators.

Permission Structure

quill.admin
├── quill.command
├── quill.info
├── quill.help
├── quill.script.load.global
├── quill.script.load.scope
├── quill.script.reload.global
├── quill.script.reload.scope
├── quill.script.unload.global
├── quill.script.unload.scope
├── quill.script.edit.global
├── quill.script.edit.scope
├── quill.script.list
├── quill.script.list.all
├── quill.scope.create
├── quill.scope.delete.any
├── quill.scope.list
├── quill.scope.list.all
├── quill.scope.info
├── quill.scope.info.any
├── quill.scope.permission
├── quill.scope.persist
├── quill.scope.persist.any
└── quill.scope.mode

Master Permission

quill.admin (default: op)
- Grants all Quill permissions
- Intended for server administrators and trusted staff
- Includes all script and scope management capabilities

Grant to trusted admins:

permissions:
  quill.admin: true

Core Permissions

quill.command (default: true)
- Basic permission to use /quill command
- Required for any Quill interaction
- Usually granted to all players

quill.info (default: true)
- View plugin information with /quill info
- Harmless read-only information

quill.help (default: true)
- Access help system with /quill help
- Essential for user guidance

Script Management Permissions

Global Scripts

quill.script.load.global (default: op)
- Load scripts from the root scripts directory
- Global scripts affect the entire server

quill.script.reload.global (default: op)
- Reload global scripts
- Allows updating server-wide automation

quill.script.unload.global (default: op)
- Unload global scripts
- Can disable server-wide features

quill.script.edit.global (default: op)
- Edit global scripts via web editor
- Potentially dangerous, requires trust

Scope Scripts

quill.script.load.scope (default: true)
- Load scripts within scopes you own
- Safe for regular players (limited by scope permissions)

quill.script.reload.scope (default: true)
- Reload scripts in your own scopes
- Allows iteration during development

quill.script.unload.scope (default: true)
- Unload scripts in your own scopes
- Stop your own automation

quill.script.edit.scope (default: true)
- Edit scripts in your own scopes
- Enables player creativity

Note: Scope owners automatically have these permissions for their own scopes, regardless of explicit permission grants.

Script Listing

quill.script.list (default: true)
- List scripts you have access to
- Shows global scripts and your scope scripts

quill.script.list.all (default: op)
- List all scripts on the server
- Includes scripts in other players' scopes

Scope Management Permissions

quill.scope.create (default: op)
- Create new scopes
- Define regions and owners
- Should be restricted to prevent scope spam

quill.scope.delete.any (default: op)
- Delete any scope on the server
- Scope owners can always delete their own scopes

quill.scope.list (default: true)
- List scopes you own
- Basic read access

quill.scope.list.all (default: op)
- List all scopes on the server
- Administrative oversight

quill.scope.info (default: true)
- View information about scopes you own
- See boundaries, mode, owner

quill.scope.info.any (default: op)
- View information about any scope
- Server-wide visibility

quill.scope.permission (default: op)
- Manage function permissions for any scope
- Scope owners can manage their own scopes

quill.scope.persist (default: true)
- Manage persistent variables in owned scopes
- Add/remove persistent storage

quill.scope.persist.any (default: op)
- Manage persistent variables in any scope
- Administrative control

quill.scope.mode (default: op)
- Change security mode (whitelist/blacklist) for any scope
- Significant security implications

Permission Examples

Regular Player Setup

Grant basic Quill access for creative players:

permissions:
  quill.command: true
  quill.info: true
  quill.help: true
  quill.script.load.scope: true
  quill.script.reload.scope: true
  quill.script.unload.scope: true
  quill.script.edit.scope: true
  quill.script.list: true
  quill.scope.list: true
  quill.scope.info: true
  quill.scope.persist: true

What they can do:
- Manage scripts in their own scopes
- View their own scopes
- List available scripts
- Cannot affect global scripts or other players' scopes

Trusted Player Setup

Grant additional scope creation ability:

permissions:
  # All regular player permissions, plus:
  quill.scope.create: true

What they can do:
- Create their own scopes
- Full management of their scopes
- Cannot interfere with other players

Moderator Setup

Grant oversight capabilities without global script access:

permissions:
  # All trusted player permissions, plus:
  quill.script.list.all: true
  quill.scope.list.all: true
  quill.scope.info.any: true

What they can do:
- View all scripts and scopes
- Monitor player automation
- Cannot modify global scripts or others' scopes

Administrator Setup

Grant full Quill access:

permissions:
  quill.admin: true

What they can do:
- Everything
- Manage global scripts
- Manage all scopes
- Full control over the plugin

Permission Inheritance

Quill's permission system respects the following ownership rules:

  1. Scope Owners: Always have script management permissions for their own scopes, regardless of explicit permission grants
  2. Operators: Have all permissions by default unless explicitly denied
  3. Master Permission: quill.admin inherits all child permissions

Security Best Practices

Principle of Least Privilege:
- Grant only necessary permissions
- Start restrictive, expand as needed
- Regular permission audits

Trust Levels:

New Players:
  quill.command, quill.info, quill.help

Regular Players:
  + scope script management
  + quill.scope.list, quill.scope.info

Trusted Players:
  + quill.scope.create

Moderators:
  + list.all, info.any

Administrators:
  quill.admin

Avoid:
- Granting quill.admin to players
- Allowing quill.script.*.global to untrusted users
- Giving quill.scope.mode without understanding security implications

Monitor:
- Who has quill.scope.create (prevent scope spam)
- Who can edit global scripts
- Who can view all scopes (privacy concerns)
