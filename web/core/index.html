Quill

A sandboxed scripting language for Minecraft event automation

Create community-run events, minigames, and custom game modes without bugging your admins. Simple, powerful, and safe.

Why Quill?

Powerful features designed for both players and admins

🎯

Event-Driven

React to player actions, block changes, and world events in real-time with simple event handlers.

🔒

Sandboxed

Scripts run in isolated scopes with physical boundaries, preventing interference and abuse.

🎮

Player-Friendly

Simple, readable syntax designed for non-programmers. If you can read it, you can write it.

🛡️

Permission-Controlled

Admins maintain full control with granular permission management per scope and function.

💾

Persistent

State and variables survive server restarts, keeping your games running smoothly.

✍️

In-Game Editing

Write, test, and debug code directly in Minecraft without leaving the game.

See It In Action

A simple arena deathmatch game in just a few lines

// Create an arena
let arena = new Scope(-50, 60, -50, 50, 100, 50)
arena.define(game_active, false)

// Start game on chat command
OnEvent(PlayerChat) {
    if chat.message == "!start" {
        arena.game_active = true
        broadcast("Game starting!")
        for player in arena.players {
            set_gamemode(player, "survival")
            give(player, "minecraft:stone_sword")
            teleport(player, 0, 65, 0)
        }
    }
}

// Handle player deaths
OnEvent(PlayerDeath) {
    if arena.game_active && in_region(player, arena) {
        broadcast("{player.name} was eliminated!")
    }
}
